# Technical Debt: IndexedDB Multiple Connections

**Discovered**: 2026-04-11
**Status**: Known Issue — Accepted

---

## Issue

Chrome DevTools → Application → IndexedDB shows multiple database entries for Hashes, Statics, Auth, Lives, Cart.

## Root Cause

**4 workers** import `Indexes` and create IDB connections:

| Context | File | Import Indexes? |
|---------|------|-----------------|
| Main Thread | `DB.js`, `Cart.js` | ✅ |
| Worker 1 | `onchain.js` | ✅ |
| Worker 2 | `offchain.js` | ✅ |
| Worker 3 | `discovery.js` | ✅ |
| Worker 4 | `update.js` | ✅ |

## Impact

- **DevTools clutter**: Hard to browse, see duplicate entries
- **No data corruption**: Data integrity preserved
- **No performance issue**: IDB handles are lightweight

## Why No Fix

1. **DevTools display ≠ actual problem** — data integrity preserved
2. **Connection handles are cheap** — lightweight browser objects
3. **No user-facing bug** — only developer UX affected
4. **Spec says per-origin** — connections managed by browser

- `src/core/Stores.js` — IDB initialization
