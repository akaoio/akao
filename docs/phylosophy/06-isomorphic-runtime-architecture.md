# Isomorphic Runtime Architecture

The project repeatedly tries to collapse the distance between environments. Browser and Node should not become two unrelated worlds with two incompatible truths.

## Main convictions

- The closer test boot is to production boot, the more trustworthy the result.
- Shared runtime architecture reduces entire categories of drift.
- Outputs should be treated as reality, not as disposable shadows of "real" code.
- Environment differences should be explicit, small, and principled.

## Why this matters

Many systems decay at their boundaries. "Dev mode," "test mode," and "real mode" become three different worlds, and confidence becomes theater. This philosophy resists that multiplication of reality.

It therefore mistrusts any proof that depends too heavily on a special room built only for proving.

## The cost

Isomorphism is not free. It forces discipline. But that discipline pays back by reducing invisible branch logic in the mind of the maintainer.

## The deeper principle

Truth should not depend too much on where the code happens to run.
