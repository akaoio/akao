# Hash Refactor + Dev Mode Optimization (2026-03-25)

**Author:** Duc
**Mô tả:** Tái cấu trúc hệ thống hash và tối ưu dev mode. Ba thay đổi chính:
1. Tách hash generation thành `build:hash` riêng — chạy 1 lần cuối thay vì 3-4 lần rải rác
2. Dev mode (`localhost`) bỏ qua hash validation — browser load file trực tiếp, không cần `.hash`
3. Dev build chỉ tạo 217 route files thay vì 27,865 — kết hợp `skipDynamic` + SPA fallback, giảm thời gian start từ ~15.6s → ~2.3s

---

## 1. Tách hash generation ra riêng (`build:hash`)

### Vấn đề cũ
Hash generation bị chạy trùng lặp ở 3 nơi: `build:core` hash toàn bộ `build/`, `build:games` hash riêng items, `build:index` hash riêng tags. Cùng file bị hash 2-3 lần → lãng phí I/O.

### Giải pháp
Tạo `builder/hash.js` — một entry point riêng chỉ chịu trách nhiệm hash. Xóa hash logic ra khỏi tất cả builder khác.

### Files thay đổi

| File | Thay đổi |
|------|----------|
| `builder/hash.js` | **MỚI** — đọc `build/` directory, tạo `.hash` file cho mỗi `.json` |
| `builder/core.js` | Xóa import và gọi `generateHashFiles` |
| `builder/index.js` | Xóa hash generation cho tags |
| `builder/games/index.js` | Xóa `hashGameItems` call |
| `builder/games/sync.js` | Xóa `hashGameItems` export |
| `build.js` | Thêm `"hash"` vào danh sách target hợp lệ |
| `package.json` | Thêm `"build:hash": "node build.js hash"` |

### Ý nghĩa
- Hash chạy **1 lần duy nhất** ở cuối pipeline thay vì 3-4 lần rải rác
- Mỗi builder chỉ lo việc của mình, không phải lo hash
- `npm run build` pipeline: `crypto → core → index → geo → hash` — hash là bước cuối, đảm bảo mọi file đã sẵn sàng trước khi hash

## 2. Dev mode bỏ qua hash validation (`DB.js` + `environment.js`)

### Vấn đề cũ
`DB.js` tự define `DEV` constant inline (`const DEV = ...`), dù `Utils/environment.js` đã có sẵn biến này. Khi dev, browser vẫn cố load `.hash` file → chậm và không cần thiết.

### Giải pháp
- `DB.js`: import `DEV` từ `environment.js`, khi `DEV === true` → `return load(path)` trực tiếp, skip toàn bộ hash validation
- `environment.js`: thêm `127.0.0.1` vào DEV check (trước đó chỉ check `localhost`)

### Files thay đổi

| File | Thay đổi |
|------|----------|
| `src/core/DB.js` | `import { DEV } from "./Utils/environment.js"` — bỏ inline constant |
| `src/core/Utils/environment.js` | `hostname === "localhost" \|\| hostname === "127.0.0.1"` |

### Ý nghĩa
- Dev mode load file trực tiếp, không qua hash → **nhanh hơn, luôn lấy file mới nhất**
- Production vẫn validate hash bình thường (cache invalidation)
- Thống nhất DEV detection về một chỗ duy nhất (`environment.js`)

## 3. Dev mode tạo ít route files hơn (`skipDynamic` + SPA fallback)

### Vấn đề cũ
`npm start` chạy `build:core` tạo **27,865 file `index.html`** — mỗi combination của 18 locales × 13 routes × items/games. Tất cả đều chứa **cùng nội dung**. Mất ~15.6s chỉ để viết file.

### Giải pháp
Hai thay đổi phối hợp:

**a) `skipDynamic` trong route generation:**
- `routes.js` thêm option `{ skipDynamic: true }` → chỉ tạo **static prefix** routes (ví dụ: `/en/showcase/index.html`, `/en/game/index.html`) mà không expand dynamic params (`[item]`, `[game]`)
- `core.js` detect `--dev` flag → truyền `skipDynamic: devMode`
- Kết quả: 217 route files thay vì 27,865

**b) SPA fallback trong dev server:**
- `dev.js` khi nhận URL không có extension (như `/en/game/arc-raiders/some-item`) mà không tìm thấy file → trả về `build/index.html`
- Fallback chỉ áp dụng cho URL không có extension (page navigation). URL có extension (`.js`, `.json`, `.css`) vẫn serve bình thường nếu file tồn tại, chỉ 404 khi file thực sự không có
- Client-side Router sẽ xử lý routing dựa trên URL

### Files thay đổi

| File | Thay đổi |
|------|----------|
| `builder/core/routes.js` | Thêm `{ skipDynamic }` parameter — khi `true`, skip dynamic route expansion |
| `builder/core.js` | Detect `--dev` flag, truyền `skipDynamic: devMode` vào `generateRoutes` |
| `dev.js` | SPA fallback: extensionless URL không match → serve `build/index.html` |
| `package.json` | `"start": "node build.js core --dev && node dev.js"` |

### Ý nghĩa
- `npm start` giảm từ **~15.6s → ~2.3s** (giảm 85%)
- Dev experience nhanh hơn nhiều — chỉ 217 files thay vì 27,865
- Production build (`npm run build`) **không bị ảnh hưởng** — vẫn tạo đủ 27,865 routes
- `build:index` cũng không ảnh hưởng — gọi `generateRoutes` không truyền `skipDynamic` → default `false`

## Tổng kết pipeline

### Dev (`npm start`)
```
node build.js core --dev → node dev.js
                │                │
                │                └─ SPA fallback cho dynamic routes
                └─ skipDynamic: chỉ 217 static prefix routes
```

### Production (`npm run build`)
```
crypto → core → index → geo → hash
                  │                  │
                  │                  └─ hash 1 lần cuối (thay vì 3-4 lần)
                  └─ đầy đủ 27,865 routes
```
