# THỰC TRẠNG KIẾN TRÚC ITEM BUILD (AS-IS)

> Cập nhật theo code hiện tại tại thời điểm **2026-03-25**.
> Mục tiêu của tài liệu này là phản ánh đúng hệ thống đang chạy, không phải đặc tả mong muốn.

## 1) Tổng quan hiện trạng

Hệ thống hiện có 2 pipeline chính liên quan item:

- `npm run build:games` (`node build.js games`)
  - Crawl dữ liệu game vào `games/<game-id>/items.json` + `games/<game-id>/images/*`.
  - Sync game-derived items sang `src/statics/items/<game-id>/<item-id>/` (YAML) và đồng thời compile sang `build/statics/items/<game-id>/<item-id>/` (JSON).
  - Có thể dùng `--prune` để xóa orphan item folders trong `src/build` theo canonical ID set mới nhất.
  - Hash subtree game output trong `build/statics/items/<game-id>/`.

- `npm run build:core` (`node build.js core`)
  - Build YAML → JSON cho **core-managed items** (1 cấp) tại `src/statics/items/<item-id>/`.
  - **Không** re-emit game-derived subtree `build/statics/items/<game-id>/<item-id>/` (ownership thuộc `build:games`).
  - Sinh pagination tổng `build/statics/items/meta.json` + `build/statics/items/<page>.json` từ **cả** core-managed + game-derived.
  - Sinh pagination theo game tại `build/statics/games/<game-id>/items/meta.json` + pages.
  - Tạo hash cho toàn bộ `build/` (trừ `geo`).

## 2) Nguồn dữ liệu item đang được xử lý

### 2.1 Core-managed items

- Nguồn: `src/statics/items/<item-id>/meta.yaml`.
- Được `build:core` build trực tiếp sang `build/statics/items/<item-id>/...`.

### 2.2 Game-derived items

- Nguồn sync: `games/<game-id>/items.json` (từ crawler) + `games/<game-id>/images/*`.
- Đích source chuẩn hóa: `src/statics/items/<game-id>/<item-id>/`.
- Đích build runtime: `build/statics/items/<game-id>/<item-id>/`.
- Ownership output runtime: `build:games` là nguồn phát sinh/chỉnh sửa chính cho subtree này; `build:core` chỉ đọc để index/pagination/routes.

## 3) Contract thực tế của `build:games`

### 3.1 Meta merge policy (đang chạy)

Trong `builder/games/sync.js`:

- Preserve:
  - `tags` nếu đã có trong file cũ.
  - `configs` nếu đã có trong file cũ (không bị ghi đè khi build lại).
  - mọi custom field không thuộc system fields.
- System fields hiện tại:
  - `id`, `game`, `type`, `images`, `icon`.
- Hành vi khi sync:
  - Luôn chuẩn hóa `id` theo folder `<item-id>`.
  - Luôn chuẩn hóa `game` theo namespace `<game-id>`.
  - `type` lấy từ normalized input hoặc fallback `"item"`.
  - Warning `unknown type` chỉ log 1 lần cho mỗi loại type (để tránh spam log theo từng item).
  - `icon` bị loại khỏi output cuối (`delete normalizedMeta.icon`).
  - `images` được ghi theo danh sách file thực tế trong thư mục `images/` của item.
  - `configs` luôn tồn tại trong canonical meta theo schema:
    - `configs.min.quantity`, `configs.min.value`
    - `configs.max.quantity`, `configs.max.value`
    - `configs.step`

### 3.2 Locale sync (đang chạy)

- Locale được chuẩn hóa về đúng shape `{ name, description }` cho toàn bộ ngôn ngữ.
- `en.yaml` được lấy từ `detailMapper(raw).locale` và luôn normalize; các locale khác seed/fallback từ `en`.
- Các field rác trong locale cũ (`images`, `meta`, `raw`, hoặc shape lồng `en: {...}`) bị loại bỏ trong quá trình sync.
- Build locale JSON (`<locale>.json`) được ghi lại theo dữ liệu đã sanitize.

### 3.3 Images sync (đang chạy)

- Ảnh nguồn lấy từ `games/<game-id>/images/`:
  - game mặc định: map `filename stem == raw.id`
  - game có custom resolver (vd: Diablo 4): map theo resolver riêng (icon id → filename)
- Ảnh được rename tuần tự theo index: `0.<ext>`, `1.<ext>`, ...
- Sau đó mirror từ `src/.../images/*` sang `build/.../images/*`.
- Có dedupe bằng SHA-256 nội dung ảnh: ảnh trùng hash sẽ không copy lại.

## 4) Cấu trúc item game hiện tại (canonical as-is)

`src/statics/items/<game-id>/<item-id>/` gồm:

- `meta.yaml`
- `raw.yaml`
- `<locale>.yaml`
- `images/0.webp`, `images/1.webp`, ... (nếu có)

`build/statics/items/<game-id>/<item-id>/` gồm:

- `meta.json`
- `<locale>.json`
- `images/*`

Ghi chú:

- `raw.yaml` được ghi đè mỗi lần sync bằng dữ liệu crawler verbatim.

## 5) Route và key item đang dùng

- Key 2 tầng đang dùng trong UI: `<game-id>/<item-id>`.
- Component item card map key 2 tầng vào path: `/item/<game-id>/<item-id>`.
- Route file đã có: `src/UI/routes/item/[game]/[item]/index.js`.
- Route item phẳng cũ vẫn tồn tại: `src/UI/routes/item/[item]/index.js`.

Kết luận route runtime hiện tại:

- Game-derived item: `/<locale>/item/<game-id>/<item-id>/`.
- Core-managed item: `/<locale>/item/<item-id>/`.

## 6) Pagination/index hiện tại trong `build:core`

- `build/statics/items/meta.json` + `build/statics/items/<page>.json`:
  - đang chứa **all items** (core-managed + game-derived) theo key chuỗi.
  - game-derived được biểu diễn dạng `<game-id>/<item-id>`.
  - không còn dùng `build/statics/items/all/*`.
- `build/statics/games/<game-id>/items/meta.json` + pages:
  - được generate từ game namespace đã discover ở `src/statics/items/<game-id>/`.

## 7) Hash ownership hiện tại

- `build:games`: hash `build/statics/items/<game-id>/` sau sync.
- `build:core`: hash toàn bộ `build/` (exclude `geo`) ở cuối build.

Hệ quả:

- Khi chạy full flow (`build:games` rồi `build:core`), hash cuối cùng vẫn do `build:core` regenerate toàn cục.

## 8) Trạng thái triển khai hiện tại (đã xác nhận)

1. ✅ Có `raw.yaml` per item game (verbatim từ crawler).
2. ✅ Image dedupe bằng hash nội dung (SHA-256 bytes).
3. ✅ Crawl gate + `--force` ở orchestrator `build:games`.
4. ✅ Enforce enum `type` với fallback `item` (warning theo từng loại type, không spam mỗi item).
5. ✅ Cấm `price|sale|currency` trong canonical meta.
6. ✅ Locale sanitize về `{name, description}` và loại bỏ `images/meta/raw` khỏi locale files.
7. ✅ Ownership tách rõ:
  - `build:games` sở hữu emit game-derived subtree `build/statics/items/<game>/<item>`
  - `build:core` không re-emit subtree này, chỉ build core-managed + index/pagination/routes
8. ✅ Pagination all-items nằm trực tiếp ở `build/statics/items/meta.json` + `build/statics/items/<page>.json` (không còn `items/all/*`).
9. ✅ Có `--prune` cho `build:games` để dọn orphan item folders.
10. ✅ `meta.configs` được đảm bảo tồn tại; nếu item đã có `configs` thì build không ghi đè.

## 9) Trạng thái thay đổi mới nhất (liên quan ảnh)

## 11) ID canonical cho game-derived items

Đã thống nhất scheme ID cho toàn bộ game-derived items tại lớp sync (`builder/games/sync.js`):

- Format: `<named-slug>-<hash5>`
- `named-slug`: slug từ tên item (`locale.name` ưu tiên, fallback `raw.name`)
- `hash5`: 5 ký tự cuối của SHA-256 trên khối `raw` (stable-stringify để đảm bảo deterministic)

Ví dụ:

- `explosive-mine-blueprint-f49d7`
- `ahavarion-spear-of-lycander-58f09`
- `anvil-i-45f38`

Lý do chọn scheme này:

- giữ URL/readability tốt (slug theo tên)
- đảm bảo uniqueness ngay cả khi `name` bị trùng nhiều lần
- không phụ thuộc format id riêng của từng game (slug string, numeric id, ...)
