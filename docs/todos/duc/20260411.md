## BUG 1: Orphan Slugs (HTTP 404)

### Hiện tượng
- Console log: `GET /statics/items/diablo-4/aegis-8c9ac/vi.json 404 (Not Found)`
- Nhiều 404 errors cho các item slugs cũ
- Trang load chậm vì retry các request thất bại

### Nguyên nhân (Root Cause)

**1. Dynamic field trong hash generation:**

Hàm `buildCanonicalItemId()` tại `builder/games/sync.js:53-67` hash toàn bộ raw data:

```javascript
function buildCanonicalItemId(raw, localeData) {
    const preferredName = localeData?.name || raw?.name || raw?.id || "item"
    const namedSlug = slugify(preferredName) || "item"
    const rawBlock = stableStringify(raw)  // ← Hash toàn bộ raw data
    const hash = sha256(rawBlock)
    const suffix = hash.slice(-5)
    return `${namedSlug}-${suffix}`
}
```

Raw data từ Wowhead bao gồm field `popularity` - một dynamic metric thay đổi theo thời gian:

```javascript
// crawler.js:20-31 - Raw item từ Wowhead
{
    id: 592242,
    name: "Aegis",
    quality: 5,
    itemTypeName: "Shield",
    playerClassNames: ["Necromancer", "Paladin"],
    slotNames: ["Off-Hand"],
    popularity: 85  // ← Dynamic! Thay đổi giữa các lần crawl
}
```

**2. Orphan folder không bị xóa:**

Mỗi lần crawler chạy với `popularity` khác:
- Crawl 1: `popularity=85` → hash → `aegis-8c9ac`
- Crawl 2: `popularity=92` → hash → `aegis-f0af7` (slug MỚI!)

Kết quả:
```
src/statics/items/diablo-4/
├── aegis-8c9ac/    ← ORPHAN (từ crawl 1)
└── aegis-f0af7/    ← ACTIVE (từ crawl 2)
```

Mặc định `build:games` có `--prune=false`, nên orphan folders không bị xóa.

**3. build:core đưa orphan vào pagination:**

`build:core` scan toàn bộ `src/statics/items/`, thấy CẢ 2 folders → thêm CẢ 2 vào pagination JSON.

**4. 404 khi browser request:**

```
Browser: GET /statics/items/diablo-4/aegis-8c9ac/vi.json
Server: Check build/statics/items/diablo-4/aegis-8c9ac/ → NOT FOUND
Response: HTTP 404
```

### Phương hướng giải quyết

**Fix ngắn hạn (đã thực hiện):**

```javascript
// builder/games/sync.js - CHỈ hash các stable fields
function buildCanonicalItemId(raw, localeData) {
    const preferredName = localeData?.name || raw?.name || raw?.id || "item"
    const namedSlug = slugify(preferredName) || "item"
    // Stable fields only - exclude popularity (dynamic metric from Wowhead)
    const stableRaw = {
        id: raw.id,
        name: raw.name,
        quality: raw.quality,
        itemTypeName: raw.itemTypeName,
        playerClassNames: raw.playerClassNames || [],
        slotNames: raw.slotNames || [],
    }
    const rawBlock = stableStringify(stableRaw)
    const hash = sha256(rawBlock)
    const suffix = hash.slice(-5)
    return `${namedSlug}-${suffix}`
}
```

**Fix vĩnh viễn:**
- Chạy `node build.js games --prune` sau mỗi lần crawl để xóa orphan folders
- Hoặc bật `--prune=true` làm default

**Commands đã chạy:**
```bash
node build.js games --game=diablo-4 --prune  # Xóa orphan folders
node build.js core                            # Rebuild pagination
```

## BUG 2: Duplicate IDB Connections

### Hiện tượng
Chrome DevTools → Application → IndexedDB hiển thị nhiều database cùng tên (Hashes, Statics, Auth, Lives, Cart).

### Nguyên nhân

**4 workers import Indexes (not all workers):**

| Context | File | Import Indexes? |
|---------|------|-----------------|
| Main Thread | threads/main.js | ❌ Statics only |
| Worker 1 | threads/onchain.js | ✅ |
| Worker 2 | threads/offchain.js | ✅ |
| Worker 3 | threads/discovery.js | ✅ |
| Worker 4 | threads/update.js | ✅ |
| qr.js | threads/qr.js | ❌ disabled |
| wave.js | threads/wave.js | ❌ disabled |
| torrent.js | threads/torrent.js | ❌ disabled |
| sql.js | threads/sql.js | ❌ no IDB needed |

Mỗi worker import `Stores.js` → mỗi cái gọi `new IDB({name: 'Hashes'})` → mỗi cái mở 1 IDB connection riêng.

4 workers thực sự dùng Indexes: onchain, offchain, discovery, update.

```javascript
// Stores.js
export const Indexes = {
    Hashes: new IDB({ name: "Hashes" }),  // ← Mỗi context tạo connection mới
    Statics: new IDB({ name: "Statics" }),
    Auth: new IDB({ name: "Auth" }),
    Lives: new IDB({ name: "Lives" }),
    Cart: new IDB({ name: "Cart" })
}

// Chỉ 4 workers thực sự dùng Indexes:
// - threads/onchain.js ✅
// - threads/offchain.js ✅
// - threads/discovery.js ✅
// - threads/update.js ✅
// Others: qr, wave, torrent disabled; sql no IDB needed
```

### Phương hướng giải quyết

**Fix lâu dài (chưa thực hiện):**
- Chỉ main thread mở IDB connections
- Workers giao tiếp với main thread qua postMessage
- Workers không import Stores.js trực tiếp

**Fix tạm thời:**
- Không cần fix vì data thực tế không bị trùng lặp
- DevTools hiển thị nhiều connections nhưng cùng origin → data nhất quán

### Cách test

1. Mở Chrome DevTools → Application → IndexedDB
2. Verify các databases: Hashes, Statics, Auth, Lives, Cart
3. Data trong mỗi database nên giống nhau (cùng origin)

## BUG 3: Dead Data in IDB/SQL

### Hiện tượng
IDB và SQL chứa data cho các slug không còn tồn tại trong `build/`.

### Nguyên nhân

**1. Data được cache khi browse:**

Khi user browse trang với slug `aegis-8c9ac`:
- IDB Hashes: `["statics","items","diablo-4","aegis-8c9ac","vi.json"]` → hash value
- IDB Statics: `["statics","items","diablo-4","aegis-8c9ac","vi.json"]` → item data

**2. Orphan slug bị xóa khỏi build:**

Sau `build:games --prune`, folder `aegis-8c9ac/` bị xóa khỏi `build/statics/items/diablo-4/`.

**3. Dead data không được cleanup:**

`DB.get()` và `update.js` không có logic xóa dead data:

```javascript
// DB.js:57-72 (Production mode)
const memory = await Indexes.Hashes.get(path).once()
const hash = await FS.load(path?.with?.(-1, path?.at?.(-1)?.replace?.(/\.\w+$/, ".hash")))
if (memory && hash && memory === hash) {
    // Cache hit - return from IDB
    const cached = await Indexes.Statics.get(path).once()
    return cached
}
// ...
const data = await FS.load(path)
if (typeof data !== "undefined") {
    await Indexes.Statics.get(path).put(data)  // ← CHỈ update khi có data
    DB.$syncToSQL(path, data)
}
// ← KHÔNG xóa khi data không tồn tại!
return data
```

Khi `FS.load()` trả về `undefined` (file không tồn tại), IDB Hashes và Statics **vẫn giữ data cũ** - không có cleanup.

### Phương hướng giải quyết

**Fix ngắn hạn:**
- User chạy `reset()` trong DevTools console để clear IDB hoàn toàn

**Fix lâu dài (chưa thực hiện):**
- Thêm logic trong `DB.get()` để xóa IDB entry khi `FS.load()` trả về `undefined`:
```javascript
if (typeof data === "undefined") {
    await Indexes.Hashes.del(path)
    await Indexes.Statics.del(path)
    return undefined
}
```

### Cách test

1. **Verify dead data:**
   ```javascript

   console.log("Total keys:", keys.length)

   // Check cho dead slug
   ```

2. **Clear dead data:**

## BUG 4: Infinite Loop (Context.set)

### Hiện tượng
- Trang load rất chậm hoặc treo
- Console có nhiều stack traces dạng:
  ```
  render → Context.set → notify → render → Context.set → notify → render...
  ```

### Nguyên nhân

**Circular dependency giữa Context.on("locale") và Router.setLocale():**

```javascript
// threads/main.js:114
Context.on("locale", ({ value: locale }) => Router.setLocale(locale.code))

// Router.js:262 - setLocale kết thúc bằng:
Context.set({ dictionary: Statics.dictionary, locale })

// LUỒNG LOOP:
// Context.set({ locale })
//   → notify("locale")
//     → Router.setLocale(locale.code)  // listener ở main.js:114
//       → Context.set({ locale })      // Router.setLocale gọi lại
//         → notify("locale")
//           → Router.setLocale(locale.code)  ← INFINITE LOOP!
```

### Phương hướng giải quyết

**Fix đã thực hiện (threads/main.js:113-117):**

```javascript
// TRƯỚC (gây loop):
Context.on("locale", ({ value: locale }) => Router.setLocale(locale.code))

// SAU (có guard):
Context.on("locale", ({ value: locale }) => {
    const current = Context.get("locale")?.code
    if (locale?.code !== current) Router.setLocale(locale.code)
})
```

Guard check ngăn không cho re-trigger khi đang set cùng locale.

### Cách test

1. **Verify fix:**
   - Reload trang
   - Trang nên load nhanh (không có infinite loop)
   - Console không có stack trace dạng `Context.set → notify → Context.set`

2. **Manual test locale change:**
   - Change ngôn ngữ trong settings
   - Verify page reload đúng cách, không loop
