TEST-DRIVEN PROGRAMMING

Project này có 1 bộ test chạy trên Nodejs, nhưng nó khá sơ xài và thực ra project này thiên về browser, nên việc test trên Nodejs thực ra không phản ánh đúng thực tế và đôi khi sẽ khó debug.

Cần một bộ test chạy thẳng trên browser, trong route /test. Nó sẽ đầy đủ chức năng như một bộ unit test suite đầy đủ. Nó cần phải test các tính năng có chứa trong /core và /UI. Mỗi lib core đều phải được test đầy đủ bao gồm các edge cases.

Khi test về webauthn cần có sự tương tác của user, nên việc này cũng đặc biệt cần lưu ý. UI phải thể hiện đầy đủ test gì, kết quả test như nào, số lượng bài test passed, tổng số bài test v.v... Nếu có thể thì làm sao đó để tái sử dụng các lib phục vụ việc test, thay vì viết riêng cho browser. Hay nói cách khác, framework test hiện tại phải hoạt động trong cả môi trường nodejs và browser. Nếu hiện tại đang dùng 1 npm package thì tôi sợ rằng không thể dùng tốt cho browser, nếu thật vậy thì ta phải viết riêng bộ test của mình.

## KẾ HOẠCH CHI TIẾT

### 1. Phân tích hiện trạng

`test.js` hiện tại **không phải unit test** của các module runtime — nó test build system (routes, i18n, hashes). Nó sẽ được giữ nguyên, không đụng vào.

Không có npm package test nào được cài (package.json không có devDependencies liên quan đến Jest/Mocha/Vitest). Đây là quyết định đúng đắn vì các framework đó đều cần bundler hoặc giả lập DOM (JSDOM) — không phản ánh đúng browser thật.

**Kết luận:** Tự viết micro test runner dùng chung cho Node.js và browser.

### 3. Test Runner: `src/core/Test.js`

Tự viết, khoảng 150 dòng, không phụ thuộc npm. API mô phỏng theo chuẩn Jasmine/Mocha để quen tay.

**API:**

```javascript
import Test from "/core/Test.js"

// Đăng ký test suite
Test.describe("States", () => {
    Test.it("notifies on change", async () => {
        const s = new States({ x: 1 })
        Test.assert.equal(s.get("x"), 1)
    })

    // Test cần user interaction (WebAuthn)
    Test.it("registers credential", async (done) => { ... }, { interactive: true })
})

// Chạy tất cả (hoặc từng suite)
const results = await Test.run()
// results = { passed, failed, total, suites: [...] }
```

**Cách hoạt động dual-mode:**
- **Node.js:** `Test.run()` → in kết quả ra console với màu ANSI (dùng `Colors.js`), exit code 1 nếu có lỗi.
- **Browser:** `Test.run()` → trả về object kết quả, route `/test` subscribe để render UI.

**Assert API:**
- `Test.assert.equal(a, b)` — strict equal
- `Test.assert.deepEqual(a, b)` — deep equal (JSON compare)
- `Test.assert.truthy(val)`
- `Test.assert.falsy(val)`
- `Test.assert.throws(fn)` — expect exception
- `Test.assert.rejects(promise)` — expect async rejection

**Test types:**
- `it(name, fn)` — auto test, chạy tự động
- `it(name, fn, { interactive: true })` — chờ user trigger, có nút "Run" riêng trong UI

### 4. Node.js integration

Tạo `test-browser.js` ở root (tương tự `test.js`) để chạy tất cả test files trong Node:

```bash
node test-browser.js            # chạy tất cả
node test-browser.js States     # chạy 1 suite
```

Thêm vào `package.json`:
```json
"test:browser": "node test-browser.js"
```

Tuy nhiên, các test cần browser API (IDB, WebAuthn, DOM) sẽ tự động **skip** khi chạy trong Node (dựa vào flag `BROWSER`/`NODE` từ Utils.js).

### 5. Route `/test` — UI Design

Refactor hoàn toàn `src/UI/routes/test/`. Khi vào route này:

1. **Import tất cả test files** (dynamic import từng file)
2. **Auto-run** tất cả non-interactive tests ngay lập tức
3. **Hiển thị kết quả real-time** khi từng test hoàn thành

**Layout:**

```
┌─────────────────────────────────────────────┐
│  Unit Tests            ● 47 passed  ✗ 2 failed │
│  ─────────────────────────────────────────── │
│  [Run All]  [Run Failed]                     │
├─────────────────────────────────────────────┤
│  ▼ States                    12/12 ✓         │
│    ✓ notifies on change                       │
│    ✓ deep equality check                      │
│    ✓ multiple subscribers                     │
│    ...                                        │
│  ▼ Router                    8/9  ✗           │
│    ✓ extracts locale from path                │
│    ✗ optional catch-all param   [see error]   │
│    ...                                        │

│    ○ register credential        [Run]         │

```

- Mỗi suite có thể collapse/expand
- Lỗi hiển thị inline: expected vs actual
- Progress bar tổng ở top
- Interactive tests có badge riêng, không chạy tự động

### 6. Danh sách test cases theo module

#### 6.1 `Events.test.js` (Universal)
- Emit và nhận event
- Multiple listeners cho cùng 1 event
- Unsubscribe bằng return value của `on()`
- `off()` không throw nếu listener không tồn tại
- Detail được wrap đúng cách `{ detail }`

#### 6.2 `States.test.js` (Universal)
- Khởi tạo với initial state
- `get()` trả về đúng giá trị
- `set()` chỉ trigger notification khi value thực sự thay đổi (deep equality)
- Multiple subscribers cùng key đều được notify
- `on()` trả về unsubscribe function hoạt động được
- `has()` / `del()` / `clear()`
- SET (global) vs MAP (path-specific) subscription
- Nested object assignment qua Proxy
- Primitive shorthand: `set("key")` = `set({ key: true })`

#### 6.3 `Router.test.js` (Universal — mock Statics)
- Parse path với locale prefix: `/en/item/some-slug`
- Extract dynamic params: `[item]` → `{ item: "some-slug" }`
- Catch-all: `[...rest]` → `{ rest: ["a", "b", "c"] }`
- Optional catch-all: `[[...rest]]` không bắt buộc
- Search params merge với path params
- Path params override search params khi trùng key
- Trailing slash normalize
- File extension strip (`.html`)
- Fallback về home route khi không match
- Locale code `zh-TW` (có dấu gạch ngang)

#### 6.4 `Utils.test.js` (Universal)
- `sha256()` — hash string cho ra hex chuẩn
- `base64UrlToHex()` / `hexToBase64Url()` round-trip
- `formatNumber(1234567.89, 2)` → `"1,234,567.89"`
- `beautifyNumber(1500000)` → `"1.5M"`, `(999)` → `"999"`
- `shorten("abcdef1234", 3, 3)` → `"abc...234"`
- `randomInt(1, 10)` — trong khoảng hợp lệ
- `randomText(8)` — đúng độ dài
- `spintax("{a|b|c}")` — trả về một trong các lựa chọn
- `encodeQuery({a: "1", b: "2"})` — URL-encoded đúng
- `clone()` — deep clone, không share reference
- `merge()` — merge không mutate original
- `logic("AND", [true, false, true])` → `false`
- `NODE` / `BROWSER` flags

#### 6.5 `Forex.test.js` (Universal)
- Self-conversion: `convert(100, "USD", "USD")` → `100`
- Bidirectional rate: nếu USD→EUR = 0.9 thì EUR→USD ≈ 1.111
- Cross-rate qua intermediary
- Invalid currency → throw hoặc return null
- `process()` merge rates không làm mất existing rates
- `filter()` loại bỏ rate = 0 hoặc NaN

#### 6.6 `DB.test.js` (Browser only — cần IDB)
- `DB.path(12345)` → `["12", "34", "5"]`
- `DB.path(100)` → `["1", "0", "0"]`
- `DB.path(5)` → `["5"]`
- Cache hit: hash match → không fetch mạng (mock fetch)
- Cache miss: hash mismatch → re-fetch và update store
- Missing hash file → fetch data anyway

#### 6.7 `IDB.test.js` (Universal — Node dùng in-memory)
- `put` / `get` round-trip
- `del` xóa đúng key
- Nested path: `get(["a", "b"])` trả về `data.a.b`
- `watch()` nhận notification khi value thay đổi
- Concurrency: multiple `put` liên tiếp không corrupt

#### 6.8 `Cart.test.js` (Browser only)
- `add()` tạo key từ SHA256(id+sku+variants)
- Thêm cùng item 2 lần → quantity tăng, không tạo entry mới
- `increase()` / `decrease()` — quantity min = 1
- `remove()` xóa đúng item
- `update()` recalculate total và quantity đúng
- `list` sort theo timestamp (mới nhất lên đầu)
- Item thiếu metadata bị remove khỏi list

#### 6.9 `UI.test.js` (Browser only — cần DOM)
- `html\`...\`` trả về TemplateResult object, không phải DOM node
- Primitive values nhúng thẳng vào HTML string (không có marker)
- Complex values (node, array, nested template) dùng comment marker
- `render(template, container)` — mount vào DOM đúng
- Array mapping: `html\`<ul>${[1,2,3].map(i => html\`<li>${i}</li>\`)}</ul>\``
- Template caching (cùng static string → cùng template instance)
- Attribute functions được call với context đúng

#### 6.10 `Context.test.js` (Browser only — cần localStorage)
- Theme: localStorage override system preference
- Theme fallback về first theme trong Statics nếu không có gì
- `setTheme()` cập nhật localStorage + `document.documentElement.dataset.theme`
- Fiat: `setFiat()` persist + notify subscribers
- Fiat invalid → default về site default

#### 6.11 `Access.test.js` (Browser only — cần localStorage)
- `getWallet()` / `setWallet()` localStorage round-trip
- Malformed JSON trong localStorage → default `{id: 0}`
- Wallet/avatar không update nếu chưa authenticated
- Default wallet/avatar = `{id: 0}`

#### 6.12 `WebAuthn.test.js` (Browser — Interactive)
- [INTERACTIVE] Register new credential → trả về credential object hợp lệ
- [INTERACTIVE] Authenticate với credential đã register → trả về signature
- [INTERACTIVE] PRF extension: derive key từ credential
- Verify key pair deterministic (cùng credential → cùng key pair)

### 8. Constraints & Notes

- **Không cần bundler** — Test.js và tất cả test files là ES modules thuần, import qua `/core/` path
- **Không mock browser API** ở browser tests — test thật trên API thật (IndexedDB thật, crypto thật)
- **Node skip** — test file tự detect `BROWSER`/`NODE` và skip các test không applicable, không throw
- **Build required** — sau khi thêm `Test.js` và `tests/` vào `src/`, chạy `npm run build:core` để copy vào `build/`
- **test.js gốc giữ nguyên** — `npm test` vẫn chạy build system test như cũ
