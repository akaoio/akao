# TÁI CẤU TRÚC CÁCH HASH CÁC FILE DỮ LIỆU HỆ THỐNG

Hiện nay có 2 hệ thống build, và mỗi hệ thống đều tạo ra các file dạng .hash đồng cấp với file json đích, và các file với tên dạng <hash> và không có đuôi file. 

- build.js: tạo các file build/statics/*.hash và các file <hash> trong build/statics/hashes
- geo.js: tạo các file geo/data/*.hash và các file <hash> from geo/hashes

npm run build sẽ cố gắng copy các file từ geo/data sang cho build/statics/geo, và geo/hashes sang build/statics/hashes (cùng chứa các file <hash> được tạo bởi build.js)

# HƯỚNG BUILD TRÊN TẠO RA CÁC VẤN ĐỀ:

- Vấn đề 1. Bản thân folder geo/hashes/ và build/statics/hashes sẽ chứa quá nhiều file (tận gần 40 triệu file) nên IDE khi mở, hoặc File Explorer khi mở sẽ bị treo và crash

=> Giải pháp: Bỏ luôn không tạo geo/hashes/ và cũng không tạo build/statics/hashes/ ở cả build và geo:build

- Vấn đề 2. Hiện nay có sự chồng chéo lẫn lộn về chức năng: npm start, npm run dev, npm run build, npm run geo:build. npm run build cũng xử lý các thông tin thuộc geo, và geo:build dường như cũng xử lý thông tin thuộc build.

=> Giải pháp:
  - Bỏ npm run dev và dev.js, ta sẽ chỉ cần npm run build và npm run geo:build
  - Đối với npm run build: vẫn theo dõi các file như trước kia, khi có sự thay đổi thì ta build lại. Nhưng build không được phép xóa build/geo/ trong bất kỳ trường hợp nào. Nó chỉ được tác động tác các file trong build/ nhưng ngoài build/geo/
  - Đối với npm run geo:build: thay vì tạo các file trong geo/data/, ta tạo thẳng trong build/geo/

- Vấn đề 3 RẤT QUAN TRỌNG. Hiện nay ta đang dùng build/statics/geo để lưu các thông tin về geo, nhưng nó dễ gây xung đột giữa geo.js và build.js.
=> Giải pháp: geo.js phải ghi dữ liệu vào build/geo/ thay vì build/statics/geo/
