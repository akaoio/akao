## 1. `/pools` Route Header Enhancement — Visual Density & Information Architecture

The current pools page header feels sparse against the full background canvas. We need to increase visual density and information value while maintaining the cyberpunk aesthetic. Defer styling principle decisions until design review, but establish the content and layout framework now.

**Core Problem:**

- Header occupies minimal vertical space with only title, live dot, and pool count
- Background feels empty and underutilized
- Missing context about the scale and activity of the pools ecosystem
- No visual hierarchy to guide user attention

**Proposed Information Additions:**

- **Aggregate TVL**: Total value locked across all displayed pools (with fiat conversion)
- **Active Chains**: Count of blockchain networks with live pools
- **DEX Distribution**: Breakdown of pools by DEX protocol (Uniswap V2/V3, SushiSwap, etc.)
- **Time Window**: "Last 24h" or "Live" activity indicator with volume metrics
- **Pool Health**: Percentage of pools with recent activity vs stale pools

**Layout Options to Consider:**

- **Hero Section**: Expand header into a full-width hero with gradient background and key metrics in a grid
- **Metric Cards**: Small cards floating over background with animated counters
- **Background Integration**: Subtle geometric patterns or data visualizations in the background
- **Progressive Disclosure**: Basic metrics visible, advanced stats on hover/expand

**Technical Considerations:**

- Data sources: Aggregate from existing `Lives.pools` data structure
- Performance: Cache aggregated metrics to avoid recalculating on every render
- Responsiveness: Metrics should stack/reorganize on mobile breakpoints
- Accessibility: Ensure high contrast for metric text against background

**Next Steps:**

1. Audit current header space usage and background utilization
2. Prototype 2-3 layout variations with placeholder data
3. Define data aggregation logic in pools route logic.js
4. Establish CSS custom properties for metric styling consistency

**Open Questions:**

- Should we include trending indicators (pools with highest volume changes)?
- How to handle fiat currency conversion for aggregate TVL?
- Whether to add mini-charts or sparklines for key metrics?
- Integration with existing theme system (light/dark/cyberpunk variants)?
